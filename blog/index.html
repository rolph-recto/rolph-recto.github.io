Embedding and Extending Go in Haskell

Dec 18, 2017   -   #cs   #pl  
Here’s a fun question: what would Go look like embedded inside of Haskell? By “Go” here I don’t mean the full language itself but some kind of core calculus with the features that people really like about Go: namely, its synchronization primitives. This core calculus should at least have the ability to spawn new goroutines and the ability to pass data between goroutines using channels. By “embedding” a language in Haskell, I mean that I want to re-use Haskell’s constructs for most things and only write what is specific to the language.
